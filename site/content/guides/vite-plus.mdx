Nub and Vite+ overlap on the everyday Node workflow — running scripts, running package binaries, and managing Node itself — but their larger goals diverge. Nub is a full package manager and a better runtime on top of stock Node; [Vite+](https://viteplus.dev) is the integrated frontend toolchain from the Vite team.

| Functionality | Nub | Vite+ |
| --- | :-: | :-: |
| Script runner (`npm run` equivalent) | ✅ | ✅ |
| Bin runner (`npx`) | ✅ | ✅ |
| Node.js version manager (`nvm`) | ✅ | ✅ |
| TypeScript runner (`tsx`) | ✅ | ❌ |
| Package manager | ✅ | ❌ — passthrough |
| Package manager shims (`corepack`) | ✅ | 🟡 — npm only |
| Dev server | ❌ — bring your own | ✅ — Vite |
| Bundler | ❌ — bring your own | ✅ — Rolldown |
| Formatter | ❌ — bring your own | ✅ — Oxfmt |
| Linter | ❌ — bring your own | ✅ — Oxlint |
| Type checker | ❌ — bring your own | ✅ — tsgo (embedded) |
| Test runner | ❌ — bring your own | ✅ — Vitest |

## Running TypeScript

Running a file with Nub adds tsx-style TypeScript execution — full syntax transpilation and `tsconfig`-aware resolution — plus polyfills and unflagged APIs like `Worker`, all still executing on the stock `node` binary. Vite+ covers TypeScript through its build and packaging tools; it does not offer a tsx-style path for running a `.ts` file directly with project-aware semantics.

This is transpilation, not type-checking. Nub supports the full TypeScript syntax surface — `tsconfig` paths, decorators, enums, parameter properties, extensionless imports — but it does not type-check. For that, bring your own type checker.

## Package management

Nub is a full package manager, powered by the embedded [aube](https://github.com/jdx/aube) engine — it resolves the dependency graph, links `node_modules`, and round-trips whatever lockfile your project already uses. Vite+ is a [passthrough](https://viteplus.dev/guide/env) to an existing package manager: in a pnpm project, `vp install` runs pnpm's install.

## The frontend toolchain

Vite+ bundles the whole frontend toolchain — the dev server, bundler, formatter, linter, type checker, and test runner named in the table above — as one tested stack. Nub intentionally leaves those concerns to whatever project-local tools you choose through `devDependencies`.
