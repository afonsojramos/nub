Nub 0.5.0 ships `nub init` — one command scaffolds a TypeScript project that runs and typechecks immediately:

```console
$ nub init -y
created my-app
  package.json
  tsconfig.json
  index.ts
  .gitignore
  README.md
  .git/ initialized

devDependencies:
+ @nubjs/types@0.5.0
+ @types/node@26.1.1
+ typescript@7.0.2

next: nub index.ts
```

## nub init

- **TypeScript 7 by default.** The scaffold pins `typescript@^7` — the native compiler — alongside `@types/node@^26` and `@nubjs/types`. Nub transpiles at run time, so `tsc` exists purely to typecheck (`noEmit`) and the project has zero runtime dependencies.
- **Node types without `"dom"`.** `lib` stays `["es2024"]`; the globals Nub polyfills (`Worker`, …) are typed by `@nubjs/types`, so `window` and `document` never leak into a Node project.
- **Real `.ts` imports.** `verbatimModuleSyntax` and `allowImportingTsExtensions`: import specifiers say `./thing.ts` and resolve exactly as Node and Nub do.
- **Strict from the first file.** `strict` plus `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes`, `nodenext` resolution, `es2024` target.

In a terminal, init asks three questions — name, TypeScript or JavaScript, git — each with a sensible default; `-y` skips them, non-interactive environments auto-default, and existing files are never overwritten without `--force`. The [init docs](/docs/init) have the full surface.

## nub create

`nub create <template>` resolves the ecosystem's `create-*` convention (`vue` → `create-vue`) and runs the scaffolder. As of this release the scaffolder can tell Nub is calling — [#529](https://github.com/nubjs/nub/pull/529) exports the `npm_config_user_agent` these tools read — so scaffolders with Nub detection respond with Nub commands. Working today:

```sh
nub create vue my-app
nub create vite my-app
nub create nuxt my-app
```

```console
$ nub create vue my-app --default

└  Done. Now run:

   cd my-app
   nub install
   nub run dev
```

Detection PRs are open across the ecosystem — track them to see your framework land: [create-next-app](https://github.com/vercel/next.js/pull/95331), [create-hono](https://github.com/honojs/create-hono/pull/124), [create-t3-app](https://github.com/t3-oss/create-t3-app/pull/2237), [create-payload-app](https://github.com/payloadcms/payload/pull/17163), [create-strapi-app](https://github.com/strapi/strapi/pull/26864), [create-medusa-app](https://github.com/medusajs/medusa/pull/15898), [create-qwik](https://github.com/QwikDev/qwik/pull/8786), and [std-env](https://github.com/unjs/std-env/pull/197). Already merged upstream: [create-vue](https://github.com/vuejs/create-vue/pull/1066), [Nuxt](https://github.com/nuxt/cli/pull/1360), [solid-cli](https://github.com/solidjs-community/solid-cli/pull/77), [react-router](https://github.com/remix-run/react-router/pull/15276), [create-cloudflare](https://github.com/cloudflare/workers-sdk/pull/14499), and [package-manager-detector](https://github.com/antfu-collective/package-manager-detector/pull/72). create-vite needs no patch — it reads the user agent directly.

## nubx returns to npx parity

`nubx <name>` no longer tries file and script resolution first — it resolves the `node_modules/.bin` chain and falls back to the registry on a miss, exactly as npx does. Files are `nub <file>`, scripts are `nub run <script>`. ([#525](https://github.com/nubjs/nub/pull/525))

## Package manager

Top-level `packageExtensions` in the root manifest is now honored in Nub-native projects — it was already supported in pnpm- and npm-compatibility mode ([#507](https://github.com/nubjs/nub/pull/507), [#509](https://github.com/nubjs/nub/pull/509)). Alongside it, a batch of correctness fixes:

| Area | What changed | PR |
| --- | --- | --- |
| `nub update` | Accepts dist-tag specs (`nub update pkg@beta`) | [#516](https://github.com/nubjs/nub/pull/516) |
| Hoisted linker | Workspace-spanning hoisted planning for `nodeLinker=hoisted` | [#511](https://github.com/nubjs/nub/pull/511) |
| `approve-builds` | Runs the approved build scripts instead of only recording approval | [#503](https://github.com/nubjs/nub/pull/503) |
| `dedupe` | Workspace link packages no longer appear in the dedupe diff | [#502](https://github.com/nubjs/nub/pull/502) |
| `publish` | Scope is preserved in the publish attachment key and tarball URI | [#478](https://github.com/nubjs/nub/pull/478) |
| Registry traffic | Dist-tag resolution is cached and metadata fetches slimmed | [#505](https://github.com/nubjs/nub/pull/505) |

## Fixes and platforms

- `nub watch` forwards an explicit `--env-file` to the watched Node so restarts re-read it ([#510](https://github.com/nubjs/nub/pull/510)).
- A macOS group-reaper watcher prevents SIGKILL of Nub's process group from orphaning the workload ([#504](https://github.com/nubjs/nub/pull/504)).
- `nub upgrade` supports the self-owned channel on Windows ([#506](https://github.com/nubjs/nub/pull/506)).
- Node installs stream extraction during download and overlap the checksum fetch ([#508](https://github.com/nubjs/nub/pull/508)).
- Source builds work on Android/Termux without a JVM ([#501](https://github.com/nubjs/nub/pull/501)).
